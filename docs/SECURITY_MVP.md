# SEC-001（MVP）达成声明

对应 SRS **SEC-001** 与架构文档 **§12** 在 **MVP / RC** 阶段的**分阶段落地**：本文档说明当前版本已实现项、已知边界与后续硬化方向，**不替代**正式安全评审或威胁建模。

---

## 1. MVP 范围内已落实

| 目标 | 说明 |
| --- | --- |
| 默认本机处理 | 不静默上传用户文档至公网（与 SRS §1.9、NFR-011 一致） |
| 扩展插件来源 | 仅**随包内置**与**运维预置**的 `plugins_extra/` 目录；应用内不提供用户 zip/目录安装 |
| 插件元数据 | `plugin.toml` 解析与启用/禁用；内置与扩展分离 |
| 错误与日志 | 统一错误体；文件名脱敏（`input_filename_hint` 等） |
| 依赖可追溯 | `Cargo.lock`、Python 锁文件策略见实施方案 DEP-004 |

---

## 2. 已知边界（须在发布说明中可见）

| 项 | 说明 |
| --- | --- |
| 扩展插件完整性 | MVP 以目录复制 + TOML 校验为主；**未**强制 SHA256/签名门禁（架构 §14.3 后续项） |
| 路径穿越 | 对 `plugins_extra/` 下目录扫描与文件访问遵循宿主路径策略（见实现） |
| 供应链 | Pandoc 随包或脚本下载；许可证与合规见 ADR-002 |

---

## 3. 后续（非 RC 阻塞）

- 扩展包校验（哈希/签名）、更细粒度 ACL
- 插件更新通道与回滚策略（与 MAINT-001 等衔接）
- 正式威胁建模与渗透测试排期
